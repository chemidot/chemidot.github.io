Hexo on Windows 10

2017-09-30
Hexo

Hexo

github 서버를 이용하는 마크다운 블로그 프레임입니다. 지금 사용하는 블로그도 hexo를 사용하여 만들어져 있는데요. 티스토리, 워드프레스를 거쳐서 hexo로 정착하였습니다. 티스토리는 수년간 사용할 정도로 큰 문제는 없었어요. 그런데, 무슨 이유에서인지 자체 백업기능을 삭제해버려서 살짝 걱정이 되더라고요. 그래서 중간에 워드프레스로 이전을 했었습니다. 그런데, 워드프레스 자체는 매우 훌륭한 서비스이지만, 별도의 이용요금 지불없이 무료서비스를 이용하다보니 중국내 접속속도가 느리고, 테마나 폰트 수정에 있어 자유도가 떨어지더군요. 글자크기를 마음대로 조절할 수 없어서 가독성이 떨어지는 것이 가장 큰 문제였습니다. 이후로 마크다운을 지원하면서 워드프레스 마이그레이션을 동시에 지원하는 서비스를 찾아보기 시작했고, 다행히도 hexo에서 wordpress-to-hexo migration 툴을 제공하여 (비록 시간은 많이 들었지만) 큰 어려움없이 이전을 진행하였습니다.

Hexo installing on Windows 10

hexo는 티스토리나 워드프레스처럼 가입만하면 이용할 수 있는 서비스는 아닙니다. 약간의 수고로움이 필요한 설치과정을 거쳐야 하는데요. 그렇지만 엄청 어려운것도 아니에요. hexo 공식사이트에 있는 메뉴얼 차근차근 읽어보면 그리 어렵지는 않습니다. 저의 경우 hexo로 마이그레이션하기로 결정하기 이전에 이와 유사한 jekyll을 설치해보았는데요. 둘다 사용해본 입장에서 hexo는 jekyll보다도 훨씬 간단합니다.

그럼에도 설치과정을 다시 정리하는 이유는, hexo 사이트에 나와있는대로 차근차근 진행해도 에러나 경고와 같은 사소한 문제들이 과정과정마다 발생했기 때문입니다. 초보자 입장에서 이런 문제를 해결하는 방법들이 나와있는 곳을 찾기가 쉽지는 않더라고요. 그래서, 제가 윈도우10에 설치하는 과정에서 생긴 문제들을 정리하고 보완하여, 최대한 문제가 생기지 않도록, 그리고 누구나 설치하여 운용할 수 있도록 순서대로 정리를 해보려고 합니다.

설치과정

1.github 가입

2.reposotory 생성 : 2개의 repository를 생성하며, 이때 repositiry의 이름은 아래와 같이 설정합니다.

​ 1) 블로그용 repository : example.github.io

​ 2) 블로그 설정파일 저장용 : hexo-archives

3.git와 node.js 설치 : 사용하는 OS에 따라 적절한 파일을 다운로드 받아 설치하며, 아래는 windows용 파일 링크입니다.

​ 1) git : git 다운로드 경로 클릭

​ 2) node.js : nodejs.org 다운로드 경로 클릭

4.hexo 설치

​ 1) 윈도우 작업표시줄에 있는 돋보기 모양의 윈도우 검색 버튼을 누르고 ‘git bash’를 입력하여 터미널 실행

​ 2) 터미널 창이 뜨면 아래의 명령을 입력하여 hexo 설치

​ 3) 이 과정에서 표시되는 경고문은 무시해도 관계없습니다. (해당 이슈에 대하여 npm 다운그레이드를 하면 해결된다는 방법이 우세하고, 그밖에도 다양한 해결책을 찾아볼 수 있네요)

5.hexo 블로그 루트 폴더 생성, 초기화 및 플러그인 설치

그리고, _config_yml 파일을 수정하여 플러그인 설정해줍니다.

6.로컬에서 페이지 확인

테마적용 : hexo-theme-hueman

1.블로그 루트 폴더(c:\hexo-blog)에서 테마 다운로드 (c:\hexo-blog\themes 에 hueman폴더 생성)

2.환경설정 파일 수정 (c:\hexo-blog\config.yml)

3.테마 환경설정 파일 생성

테마 폴더 안에 있는 _config.yml.example의 이름을 _config.yml로 수정 (※블로그의 _config.yml 가 아님!)

4.테마에서 적용하는 검색기능 활용을 위해 hexo-generator-json-content 설치

테마 커스터마이징 : themes/hueman/_content.yml 수정

1.메뉴 (선택사항) : 글을 포스팅할때 front-matter(포스트 첫머리의 제목, 생성일자 등이 들어가는 부분)에 카테고리를 추가하면 자동으로 추가되므로, 추가/수정을 하지 않아도 됨

  • 단, 필요한 것을 추가하거나 수정하려면,

    menu:
    Home: /hexo-theme-hueman/
    About: /hexo-theme-hueman/about/index.html # 상대경로

    Github: https://github.com/ # 절대경로

2.About.me

  • /about/index.html 생성해야 함
  • 혹은 about.me 사이트를 이용해서 프로필을 만들고 해당 url을 추가해도 됨

3.로고

  • public 밑에 있는 img 폴더는 root폴더명/img라는 이름으로 접근
  • 혹은 웹의 이미지url도 가능

4.테마색깔

5) syntax highlighter

6.사이드바 위치

7.썸네일 : 포스트의 썸네일 기능

  • 썸네일 사용시 포스트 작성시 앞쪽에 자동삽입되는 front-matter 부분에 경로 추가

8.favicon : url 앞에 붙는 작은 아이콘으로 png파일

  • 바뀌지 않는 경우에는 브라우저 파비콘 파일 폴더의 파일을 삭제
    \AppData\Local\Google\Chrome\User Data\Default

9.소셜링크

10.위젯 : 사이드바에 추가할 위젯 설정

11.링크 : 위젯에서 추가한 링크 사용시에 url추가

12.검색 : 블로그내의 검색기능으로 기본제공되는 InsightSearch 사용

13.댓글 : 기본적으로 Disqus 제공. Disqus 가입 후 아이디 확인하고 _config.yml에 입력

14.공유 : 포스트 공유 기능으로 스타일을 선택

15.플러그인

16.폰트

  • 본문 폰트 크기 변경 : _variables.css 파일 수정
  • 본문 제목 크기 변경 : /_partial/article.styl 파일 수정

포스팅

새로운 글 포스팅하기

혹은 typora와 같은 마크다운 편집기를 사용하여 md확장자를 가진 파일을 만들어 _posts 폴더에 업로드합니다.

로컬작업 내용을 github 도메인으로 배포(push)

이때, deploy후에도 페이지 업데이트가 안된다면, 페이지 clean 후 재배포합니다.

배포시 발생하는 에러

1.TypeError: Cannot read property ‘exports’ of undefined : code highlight가 안되는 언어를 쓴 경우 에러 발생

해결방법 : 테마에서 지원 가능한 code highlight를 사용 (예를들어, code block 작성시 사용언어로 arduino를 지정하면 위 에러가 발생하므로, arduino 대신 cpp나 그밖의 지원가능한 언어를 지정하여야 합니다.)

2.워드프레스 대시보드에서 백업파일(.xml)을 다운 받습니다.

“Tools” → “Export” → “WordPress”

3.백업파일을 _posts폴더에 복사한 후, 마이그레이션을 진행합니다.
